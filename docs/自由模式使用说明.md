# 世界引擎自由模式使用说明

自由模式用于在内置十二模块之外，按当前世界观创建更贴合的推演结构。它可以完全使用自定义模块，也可以混合引用少量内置模块。

## 生成预设

1. 打开世界引擎预设面板，点击「从设定生成」。
2. 在生成设置里勾选要参与生成的世界书条目。关闭的条目不会参与生成；如果面板里显式只勾选现代校园条目，仙侠条目不会被读取。
3. 选择生成方式：
   - 「十二模块生成」：沿用内置十二模块，可通过 AI 自动裁剪关闭不相关模块。
   - 「自由生成」：让 AI 自行决定 modules[]，可生成纯自定义模块，也可混合引用内置模块。
4. 选择模块数量：
   - 「委任 AI 决定」：AI 按世界观自行决定模块数量。
   - 「用户指定」：要求最终启用模块数恰好为 N。十二模块生成会用 disabledModules 反推启用数；自由生成会校验 modules[] 启用数。
5. 可选勾选「同时读取当前角色卡描述」。
6. 点击「开始生成」。

## 手动编辑自由模块

在「模式与自由模块」里可切换 classic / free。

- classic：仍使用内置十二模块，下方原模块开关继续生效。
- free：显示自由模块编辑器。

自由模块编辑器支持：

- 新增、删除、上移、下移模块。
- 从内置引入模块，形成 mixed/free 预设。
- 编辑自定义模块的 name、field、container、itemKey、rules。
- 复用 schema 行编辑器编辑 fields。
- 编辑 display：cards、table、keyvalue、list。
- 启用 mechanics：dice、stages、verdicts，并可插入模板。

保存前会校验 id、字段名、重复模块、内置字段冲突、array itemKey、机制 JSON 和基础参数形状。校验失败时不会保存。

## 模块结构要点

自定义模块常用字段：

- `id`：模块 ID，英文标识符。
- `kind`：`custom` 或 `builtin`。
- `container`：`array`、`object`、`scalar`、`none`。
- `field`：写入世界状态的字段名，不能与内置字段冲突。
- `fields`：array/object 模块的输出字段定义。
- `rules`：写给推演 AI 的更新规则。
- `display`：UI 渲染配置。
- `mechanics`：可选机制配置。

内置 state 字段不要用于自定义模块：`events`、`factions`、`worldTrends`、`winds`、`economy`、`reputation`、`enemies`、`influenceChain`、`regionalIncident`、`blackbox`、`world_digest`。

## 导入导出与存档

新预设会带 `schemaVersion: 2`。旧预设导入时会自动迁移为 classic，并补齐默认结构。

自由预设导出时会保留 `mode`、`modules`、`mechanics`。如果当前对话存在对应自定义模块 state，也会随导出 JSON 带上；导入后会恢复到当前对话的自定义模块 state 槽。

删除自由模块后，旧模块留下的孤儿 state 不会参与推演；下次加载/保存 custom module state 时会自动清理。

## 验证建议

- 生成后先看「模式与自由模块」确认模块数量和模块类型符合预期。
- 运行一轮推演，确认新模块能出现在世界状态和 UI 面板中。
- 对跨风格世界书，先显式勾选条目再生成，避免把无关世界观混入 prompt。
