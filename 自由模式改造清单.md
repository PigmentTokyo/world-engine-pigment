# 世界引擎 · 自由模式改造清单

> 目标：在现有十二模块系统之外，增加「自由模式」。
> 决策：① 通用模块**保留可配置机制**（骰子/阶段/裁决均做成 config 驱动）；② classic 与 free **允许混合**（自由模式可掺入内置模块，复用其机制）。
> 架构原则：**一个引擎 + 模式开关 + 模块描述符抽象**，不做两份平行引擎。

## 图例 / 用法
- `[ ]` 未开始 → 干完改成 `[x]`。
- 🔒 = 不可回归项：完成后必须验证 **classic 模式行为与改造前一字不差**。
- ⚠️ = 高风险/大改动，建议单独分支 + 单独提交。
- 每个阶段最后都有「验证」项，过不了不许进下一阶段。

### 6.1 生成时的模块选择与数量策略（用户追加需求）
> 目标：生成预设时，让 AI 不再永远套满 12 个模块，而是按世界观挑选/裁剪；并允许用户控制模块总数。

- [ ] **内置模块裁剪**：classic/混合生成提示词增加指令，让 AI 分析世界书后判定**哪些内置模块与本世界相关**；不相关的写入 `disabledModules`（= 不生成字段、不注入规则、面板不显示），而非永远全开
- [ ] 生成组装：把 AI 返回的 `disabledModules` 落地到预设（现状生成完全不产出此字段，需补透传 + 校验）
- [ ] **模块数量策略**控件：生成面板加「模块数量」选择 —— `委任 AI 决定` / `用户指定 N 个` 二选一
- [ ] 提示词联动：选「指定 N 个」时约束 AI 产出**恰好 N 个**模块（自由模式=N 个描述符；混合=内置保留数+自定义数合计 N）；选「委任」时让 AI 自行决定数量与构成
- [ ] 结果校验：实际模块数与用户指定值一致；不符时给重试/补足/裁剪提示，不静默接受
- [ ] 与「生成方式」联动：在十二模块上生成时该策略作用于「裁剪/保留哪些内置」；自由生成时作用于「造几个模块」；混合时两者都管
- [ ] 验证：分别测 ①AI 自决数量 ②指定 5 个 ③混合（保留 3 内置 + AI 增 N 个），生成结果模块数与设定一致且可直接推演

## 涉及文件速查
- `presets.js`：mode / modules / 机制 normalize / 自由生成 / 导入导出

## 风险备忘
