# 世界引擎 · 自由模式改造清单

> 目标：在现有十二模块系统之外，增加「自由模式」。
> 决策：① 通用模块**保留可配置机制**（骰子/阶段/裁决均做成 config 驱动）；② classic 与 free **允许混合**（自由模式可掺入内置模块，复用其机制）。
> 架构原则：**一个引擎 + 模式开关 + 模块描述符抽象**，不做两份平行引擎。

## 图例 / 用法
- `[ ]` 未开始 → 干完改成 `[x]`。
- 🔒 = 不可回归项：完成后必须验证 **classic 模式行为与改造前一字不差**。
- ⚠️ = 高风险/大改动，建议单独分支 + 单独提交。
- 每个阶段最后都有「验证」项，过不了不许进下一阶段。

### 6.1 生成时的模块选择与数量策略（用户追加需求）
> 目标：生成预设时，让 AI 不再永远套满 12 个模块，而是按世界观挑选/裁剪；并允许用户控制模块总数。

- [x] **内置模块裁剪**：classic/混合生成提示词增加指令，让 AI 分析世界书后判定**哪些内置模块与本世界相关**；不相关的写入 `disabledModules`（= 不生成字段、不注入规则、面板不显示），而非永远全开
  > ✅ classic 生成已通过 `buildModuleSelectionPrompt()` 注入 `disabledModules` 要求；自由生成 prompt 也会要求只保留真正有用的 custom/builtin 模块。
- [x] 生成组装：把 AI 返回的 `disabledModules` 落地到预设（现状生成完全不产出此字段，需补透传 + 校验）
  > ✅ classic 生成会透传并校验 `disabledModules`，保存后由 rules-loader 关闭对应规则/schema/UI。
- [x] **模块数量策略**控件：生成面板加「模块数量」选择 —— `委任 AI 决定` / `用户指定 N 个` 二选一
  > ✅ 生成设置二级菜单新增模块数量模式与 N 输入，持久化到 store。
- [x] 提示词联动：选「指定 N 个」时约束 AI 产出**恰好 N 个**模块（自由模式=N 个描述符；混合=内置保留数+自定义数合计 N）；选「委任」时让 AI 自行决定数量与构成
  > ✅ classic 用 `disabledModules` 反推启用数；free/mixed 用启用的 `modules[]` 总数，prompt 均写入恰好 N 个要求。
- [x] 结果校验：实际模块数与用户指定值一致；不符时给重试/补足/裁剪提示，不静默接受
